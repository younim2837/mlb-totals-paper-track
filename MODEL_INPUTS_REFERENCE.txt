MLB TOTALS MODEL INPUTS REFERENCE
=================================

This file summarizes the current inputs used by the production MLB totals
model.

Current model family:
- Team-split XGBoost
- One model predicts home runs
- One model predicts away runs
- Final total = home prediction + away prediction
- Both side models are trained as Dixon-Coles residual models

Important:
- Kalshi lines do NOT go into the point model
- Sportsbook lines do NOT go into the point model
- Kelly sizing does NOT go into the point model
- Those are downstream comparison and betting layers only


1. HOW TO THINK ABOUT THE INPUTS
--------------------------------

The easiest way to understand the feature set is in layers:

1. Bayesian prior layer
   Dixon-Coles team attack/defense priors and expected runs

2. Team form layer
   Recent and season-to-date scoring/prevention context

3. Game environment layer
   Ballpark, dome, weather, umpire

4. Same-day roster layer
   Confirmed lineups, lineup quality, platoon structure

5. Pitching layer
   Starter quality, workload, role context, bullpen quality and availability

The production point model currently uses 175 engineered inputs.


2. POINT MODEL INPUT GROUPS
---------------------------

A. Team identity and matchup indexing
- away_team_id
- home_team_id
- h2h_avg_total_runs

Why these exist:
- team IDs let tree models learn stable team-specific tendencies
- head-to-head average total gives lightweight matchup-specific context


B. Recent team form: 10-game windows
- home_avg_runs_scored_10g
- home_avg_runs_allowed_10g
- home_avg_total_runs_10g
- home_win_pct_10g
- away_avg_runs_scored_10g
- away_avg_runs_allowed_10g
- away_avg_total_runs_10g
- away_win_pct_10g
- combined_scoring_10g
- combined_allowed_10g
- home_offense_vs_away_pitching_10g
- away_offense_vs_home_pitching_10g

Why these exist:
- capture recent offensive quality
- capture recent run prevention quality
- capture whether a team has recently been playing in higher- or lower-scoring games
- capture directional matchup tension between one offense and the other side's prevention


C. Recent team form: 30-game windows
- home_avg_runs_scored_30g
- home_avg_runs_allowed_30g
- home_avg_total_runs_30g
- home_win_pct_30g

Why these exist:
- provide a more stable medium-horizon baseline than 10-game form


D. Season-to-date team form

- away_season_avg_allowed
- combined_season_scoring
- combined_season_allowed

- away_days_rest

Why these exist:
- anchor team quality beyond short streaks
- rest days can affect lineup freshness and bullpen usage


E. Bullpen baseline quality

- precip_mm
- humidity_pct
- dew_point_f

Why these exist:
- park factor captures the venue's run environment
- dome/retractable settings help suppress irrelevant weather influence
- weather helps explain same-day scoring conditions in outdoor games
- local first-pitch timing helps distinguish day vs night run environments

Why these exist:
- this is the main Bayesian prior layer
- lambdas and expected total provide a structured pregame scoring baseline
- attack/defense parameters help XGBoost learn where the prior should be corrected

3. WHAT THE UNCERTAINTY MODEL USES
---------------------------------

The uncertainty model uses:
- all of the same baseball/game features above
- plus one extra feature: point_prediction

Its job is to predict absolute error, which is then converted into
game-specific sigma:
- prediction_std

This lets the probability layer use different spreads for different games.


4. WHAT THE TAIL MODELS USE
---------------------------

There are two tail classifiers:
- high-tail model for P(total > 9.5)
- low-tail model for P(total < 7.5)

Each tail model uses:
- all of the same baseball/game features above
- plus one extra feature: point_prediction

These tail models do NOT replace the point forecast.
They help the probability layer express when a game has extra shootout risk or
extra dud risk.


5. WHAT IS NOT A MODEL INPUT
----------------------------

The following are not inputs to the core baseball model:

- Kalshi lines
- sportsbook totals
- Kelly sizing
- edge calculations
- displayed bet signals

Those happen after prediction and are part of evaluation / decision support.


6. WHERE EACH GROUP COMES FROM
------------------------------

Main sources:
- collect_games.py -> historical completed MLB games
- dixon_coles.py -> Bayesian team priors
- collect_team_batting.py -> rolling team batting stats
- collect_team_lineups.py / lineup_features.py -> same-day lineup features
- collect_bullpen_usage.py / bullpen_usage.py -> bullpen workload and top-arm availability
- collect_pitcher_stats.py -> pitcher game logs and starter rolling stats
- collect_umpires.py -> historical home plate umpire records
- collect_weather.py -> historical weather by venue/date
- build_features.py -> assembles the historical training matrix
- predict_today.py -> assembles the same feature family live
